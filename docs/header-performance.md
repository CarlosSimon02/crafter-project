# Header — Performance Optimizations

Quick log of what changed in the header path and why each thing helps.
Date: 2026-05-10

## TL;DR

| Metric | Before | After |
|---|---|---|
| `app.js` (raw) | 199 KB | **1.2 KB** |
| `app.js` (gzip) | 62 KB | **0.57 KB** |
| FOUC on header | Yes | None |
| Layout shift on header images | Possible | Prevented |

99.4% smaller bundle, no flash before the header appears.

## What changed

### 1. Header markup moved from React to FreeMarker

**Before:** `header.ftl` emitted only a placeholder div with `data-*` attributes. The actual HTML (logo, location, CTA, language switcher) was generated by `Header.tsx` after React mounted.

```html
<!-- old header.ftl output -->
<div data-header data-logo-src="..." data-cta-label="..."></div>
```

The browser had to download `app.js`, parse it, run it, and let React mount before the header became visible. The gap was the FOUC.

**After:** `header.ftl` renders the entire header HTML server-side. The browser paints it on the first HTML response, before any JS runs.

### 2. React removed from the header path → tree-shaken from the bundle

The Header was the only component using React. Once it became plain FTL + a small `header.ts` module (~50 lines for the dropdown), nothing else imported React. Vite's tree-shaker removed React + ReactDOM entirely.

The vanilla TS file does just three things:
- Toggle the dropdown when the trigger is clicked
- Close the dropdown on outside click or `Escape`
- Navigate to `/{newLocale}/...` when a language option is clicked

No virtual DOM, no reconciliation, no JSX runtime.

### 3. `width` and `height` attributes on every `<img>`

```html
<img src="logo.svg" width="129" height="32">
<img src="flag.svg" width="25" height="25">
```

Without these, browsers don't know the image's box size until the bytes arrive — when they arrive, the page jumps as space is allocated (Cumulative Layout Shift). With explicit dimensions, the browser reserves the slot upfront. Nothing jumps.

### 4. Fonts continue to use `font-display: swap`

This was already in `main.css`. While custom fonts (GoTymeSans, Inter) load, text renders in a system fallback. When the custom font arrives, it swaps in. So text is visible immediately rather than invisible until fonts download.

## Files involved

| File | Role |
|---|---|
| `templates/web/components/header.ftl` | Full server-rendered header markup |
| `sources/js/src/header.ts` | Vanilla TS dropdown enhancer |
| `sources/js/src/main.tsx` | Imports and runs `initHeader()` on `DOMContentLoaded` |
| `sources/js/src/components/Header.tsx` | **Deleted** |

## Pattern going forward

> FTL renders by default. Vanilla TS enhances. React only when there's genuine client state.

This keeps the bundle tiny and eliminates FOUC for static layout. If a future component genuinely needs complex state (forms with derived state, real-time data, etc.), we can bring React back — but for dropdowns, accordions, tabs, simple toggles, vanilla TS is enough.

## Possible next wins (not done yet)

- `<link rel="preload">` for the hero background image to speed up Largest Contentful Paint (LCP).
- `loading="lazy"` on below-the-fold images once we build About / Partners / Footer.
- Convert `static-assets/images/product-features-hero-hk.png` (1.5 MB) to WebP — deferred until that section is built.
- Inline critical CSS — Tailwind output is currently 21 KB; small enough that this isn't urgent.
